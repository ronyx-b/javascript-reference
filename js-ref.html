Syntax, Keywords and Declaration

Basic Syntax

// …
line comment.

block comment.

Variables Declarations

var variable = value, … declares a global scope variable, may be initialized to value, may declare an initialize a set of comma-separated variables.

let variable declares a block level variable.

const constant declares a constant value o reference to an object.

Functions

function funct(parameters, …) { …; return output } function declaration: may receive "parameters", runs containing statements when called, may "return" the specified "output" if given, else "undefined" is implied.

var funct = function functName(parameters, …) { …; return output } function expression: assigns the given function to the variable; may be named, but only referred in the expression (useful when self-recurring).

(parameters, …) => { …; return output; } arrow function notation: creates anonymous functions, if it has only a single statement then its result is taken implicitly as return. arguments functions' built-in array-like object that contains the arguments list. .name nonstandard property that holds the function's name (even named function expressions). .call(thisArg, arguments, …) calls a function and sets its "this" reference to "thisArg". .apply(thisArg, argArray) calls a function and sets its "this" reference to "thisArg", provides arguments for the functions as and array "argArray". .bind(thisArg, arguments, …) creates a new function that, when called, has its "this" keyword set to the provided "thisArg".

funct(arguments, …) function calling: invokes a defined function, may require arguments to be passed.

(functionStatement)(arguments) Immediately invoked function expression: immediately calls the function defined in the grouping operator (it is a grouping operator and an invoking operator).

Compound Data types

var array = [ values,… ] array literal declaration: declares a variable "array" initialized with "values". array[index] access the element at position "index" of the array. ...list spread/rest syntax: spreads an array to a list where a list is expected or condenses a list into an array.

var object = { property: value, …, method: function(…) {…} } object literal declaration: can contain properties, methods and other objects, declared as "key: value". delete keyword operator that deletes a property from an object.

[a, b,…] = [x, y,…]; {a, b,…} = {a:x, b:y,…} Destructuring assignment: allows you to assign the properties of an array or object to variables using syntax that looks similar to array or object literals.

Logical Structures and Loops

Flow Control: Break and Continue

break label; declares an exit of the innermost loop or the one defined by the "label".

continue label; declares a jump to the next iteration of the innermost loop or the one defined by the "label".

If Else Statements

if (condition) { …; } if condition evaluated is true, executes contained statements. else if (condition) { …; } if previous conditions are false and the one indicated is true, run script. else { …; } if all previous conditions are false, run script.

Switch Case

switch (statement) { cases… } evaluates the result of "statement" and executes the "case" that matches. case value: …; break; executes statements if "statement" is equal to "value ", ends with a "break". default: script; break; run "script" if "statement" is not equal to any defined case value.

Do While Loops

label: while (condition) { …; } executes statements while defined "condition" is true.

label: do { …; } while (condition) executes statements once and as long as "condition" is true.

For Loops

label: for (initial; condition; step) { …; } sets the "initial" statement, then executes the statements in the body while "condition" is true, executing the "step" statement at the end of each iteration.

label: for (key in object) { …; } executes statements for each "key" property in the "object", using "key" as iterator key string (name of the current parameter of the object, refer as "object[key]").

label: for (item of iterable) { …; } executes statements for each "item" in the "iterable" object (array or array-like, string) using "item" as iterator.

Error Handling, Promises and Modules.

Error Handling

try { … } defines a block of statements that might produce an error. throw expression defines an exception thrown, can be a string or an "Error" object (new Error(msg)).

catch(error) { … } defines a block of statements to handle an error produced on a "try" block, and stores it in the "error" object of type "Error()". .message property of "Error" type objects that contains the error message. .name property of "Error" type objects that contains the error name.

finally { … } optional final block of statements that will be executed weather or not the "try" block threw an error.

Promises (Asynchronous Functions)

return new Promise(function callback(resolve, reject) { …; } ) the function won't return an immediate value, but rather will either be successful and send the value in a "resolve(…)", or fail and send an error message from a "reject(…)".

resolve(resolved) function declared in promise's callback that returns a "resolved" promise.

reject(rejected) function declared in promise's callback that returns a "rejected" promise.

.then(function callback(resolved) { … }) Promise's method that takes the "resolved" value and executes the statements that would follow, they can be chained (asyncFn(…).then(…).then(…).catch(…)).

.catch(function callback(rejected) { … }) Promise's method that takes the "rejected" value and executes the statements that would follow.

.finally(function callback() { … }) Promise's method that is called at the end of a promise chain when the promise is settled, whether fulfilled or rejected.

async function asyncFn(parameters,…) { … } declares an asynchronous function that returns an implied promise, can be used inside "try …catch" blocks. await asyncStatement used only inside "async" functions, stops its execution until a resolved or rejected promise is returned from the "asyncStatement" and returns that value.

Modules

import exported as alias,… from 'module-or-file-path' declares a resource imported (static) from another module, where the resources "exported" can be: the default module "defaultExport", all "*" (can also be done as "import 'module'"), or a specific list "{ exportedN as aliasN, … }".

import('module') declares a dynamic import when called as a function and returns the module as a "Promise".

export default resource declares a "resource" (function, class, object, variable) to be exported to other modules, where only one can be set as "default" which doesn't need to be named.

Built-in Objects, Properties, Methods and Functions

Parent Object

Object(…) constructor function for new objects, equivalent to: "var newObject = {};" can be initialized with a list of comma-separated pair of values in the form of "property: value,", where value can variables, functions or other objects. .create(objectTemplate) class (static) method that creates a new object with the same properties of "objectTemplate" through inheritance (with same initial values by default, stored in ".prototype"). .getPrototypeof(object) class (static) method returns the prototype of the specified object. .toString() returns a string representing the object.

String Objects Methods

String(…) constructor function for strings, can convert from numbers.

.length property that returns the length of the string.

.charAt(index) strings method, returns character at "index".

.concat(string) receives a string to be concatenated at the end of the string it is applied.

.includes(string) returns true if the "string" argument is contained in the applied string.

.startsWith(string) returns true if the string applied starts with the given "string", otherwise false.

.endsWith(string) returns true if the string applied ends with the given "string", otherwise false.

.match(regExp) returns the matches of the applied "regExp" regular expression.

.toLowerCase() returns the string with all characters in lower cases.

.toUpperCase() returns the string with all characters in upper cases.

.trim() removes blank spaces from both ends of the string.

.replace(regExp, string) returns the string with the matches of the "regExp" replaced by "string".

.search(regExp) returns the index of the first match of the "regExp", or "-1" if not found.

.slice(start, end) returns a new string from "start" index (inclusive) until "end" index (not inclusive) if given, or until the end of the original string; indexes can be negative (from the end of the string)

.split(separator, limit) returns an array of substring from the original string divided by a "separator" string or regular expression, but until reaching a "limit" number of elements if given.

.substr(start, length) (deprecated) returns a portion of the original string, from "start" index and with a defined by "length" if given.

.substring(start, end) strings method, returns part of a string starting a "start" index (inclusive) and ending before "end" index (not inclusive, end of string if omitted).

.charCodeAt(index) string method, returns Unicode value of the character at "index".

Array Objects Properties and Methods

Array(…) constructor function for arrays, empty if no arguments, otherwise with its arguments as the values.

.isArray(argument) "Array" class (static) method that returns true if the argument is an array.

.length property of any array (and array-like objects like strings and arguments list) that returns their length.

Array Modifying Methods:

.push(element) adds the arguments to the end of the array and returns the next index.

.unshift(element) adds the arguments to the start of the array and returns the new length.

.pop() removes the last element of the array and returns it.

.shift() removes the first element of the array and returns it.

.concat(elements) returns the original array joined together with other arrays or values provided in the arguments.

.slice(start, end) returns a new array from "start" index (inclusive) until "end" index (not inclusive) if given, or until the end of the original string; indexes can be negative (from the end of the string).

.splice(start, deleteCount, elements…) changes the contents of an array from "start" index by removing (if "deleteCount" > 0) and/or adding new "elements" (if any) in place, and returns an array with the removed items (empty if none).

.fill(value, start, end) assigns "value" to the elements of the array, from "start" index (inclusive, default "0") to "end" index (not inclusive, default "array.length").

.reverse() reverses the order of the elements of an array in place (the original array).

Array Query Methods:

.indexOf(element, start) returns the index of the first element from "start" index (inclusive, default "0") that corresponds to "element", or "-1" if not found.

.lastIndexOf(element, start) returns the index of the last element from "start" index and going backwards (inclusive, default "array.length - 1") that corresponds to "element", or "-1" if not found.

.incluides(element) returns "true" if the array includes the given element, otherwise "false".

.join(string) returns a string created by concatenating all elements in the array with the given "string" delimiter.

.toString() returns a string representing the array and its elements.

.toLocaleString() returns a localized string representing the array and its elements.

Array Iterating Methods:

.forEach(function callback(element, index, array) {…}) takes a callback function as argument that is called upon each "element =" of the "array".

.map(function callback(element, index, array) { …; return mapElement; }) takes a callback function as argument that is called upon each "element" of the "array" and returns a new array out of the returned values of the callback.

.find(function callback(element, index, array) { …; return boolean; }) returns the first element for which the "callback" function returns "true".

.findIndex(function callback(element, index, array) { …; return boolean; }) returns the index of the first element for which the "callback" function returns "true".

.filter(function callback(element, index, array) { …; return boolean; }) returns a new array containing all elements for which the "callback" function returns "true".

.every(function callback(element, index, array) { …; return boolean; }) returns "true" if all of the elements in the array return "true" on the callback function.

.some(function callback(element, index, array) { …; return boolean; }) returns "true" if at least one of the elements in the array returns "true" on the callback function.

.reduce(function callback(accumulator, element, index, array) { …; return toAccumulator; } , initialValue) executes a reducer function on each element assigning its returned value to the given "accumulator", and returns the final value of it; can have an "initialValue" set for the "accumulator".

.reduceRight(function callback(accumulator, element, index, array) { …; return toAccumulator; } , initialValue) executes a reducer function on each element, going from last to first, assigning its returned value to the given "accumulator", and returns the final value of it; can have an "initialValue" set for the "accumulator".

.sort(function callback(first, second) { …; return number; }) sorts the original array, by converting to strings and comparing their Unicode value, or by result of "callback" comparing two elements and sorting "first" before "second" if it returns a "number" <= 0, or backwards otherwise.

JSON Parsing Methods

.test(string) tests the RegExp for a match in its "string" parameter and returns a Boolean.

.exec(string) executes a search for a match in a specified string and returns a result array, or "null".

Fetch API

EJ: fetch(request).then(response => response.json()).then(json => {…})

  • "string": if a string is used, the string is made by each field name delimited by a blank space, and sorts by each named field ascending ("field"), or descending if the field is preceded by a minus sing ("-field").

React Component

import React from 'react' imports the "React" component, which allows to export Component functions as a React Component class ("React.Component")

function CustomComponent(props) { …; return (<>HTMLcode</>); } Functional Component: defines a React Component as a function that returns HTML and other components wrapped inside a single HTML element. May include a "props" object as an argument that contains variables passed as attributes of the component ("<CustomComponent attr=value />" allows to use "value" inside the component as "props.attr"). {statement} inside React components (JSX), embedded data and JavaScript statements are wrapped in "{ … }".

export default CustomComponent exports the component to be used by other components in react as an element "<CustomComponent />".

class CustomComponent extends React.Component { constructor() {…}; render() { …; return (<>HTMLcode</>); } } Class Component: Alternatively, we can define a React Component as an ES6 class that extends "React.Component", a class implementing an optional "constructor()" function and a required "render()" method, and in which the "props" object can be accessed through "this.props" and state variables through "this.state".

Rendering Data Syntax

{condition && <>HTMLcode</>} use the logical AND ("&&") operator to render a section "<>HTMLcode</>" only if "condition" evaluates to true.

{condition?(<>RenderOnTrue</>):(<>RenderOnFalse</>)} use the ternary operator ("?:") to have different renders for a "condition" if it's true or false.

{array.map((elem, index) => (<htmlElem key={index}>…</htmlElem>))} when rendering data iterating through a collection, each element needs to have a unique "key" attribute on the parent "HTMLelem", which can be the callback's "index" argument or another unique ID value.

React "Hooks"

import { useState, useEffect } from 'react' imports the "useState()" and "useEffect()" functions from React to work with dynamic data rendering.

const [state, setState] = useState(initialState) accepts a parameter that allows to set the initial value of a state and returns an array consisting of the current value of the "state" itself and a function "setState" to update it, causing the component to be rendered again. setState(newValue|callback) it receives either a "newValue" or a "callback" function to set the new value, which itself receives the previous value and returns the new one: "function callback(prewValue) { …; return newValue; }".

useEffect(callback, [dependencies]) accepts a callback function to be executed when the component is mounted and whenever any of variables of the optional dependencies array is modified. The callback may return another callback to be executed when unmounted.

React Router

import { BrowserRouter } from 'react-router-dom' imports the "BrowserRouter" component, which needs to wrap the "<App />" component on the "index.js " file, to enable routing.

import { Switch, Route, Redirect, Link } from 'react-router-dom' imports the routing components to wire the routing, with "Switch" and "Route" being used inside the "App" component.

import { useParams, useLocation, useHistory, useRouteMatch } from 'react-router-dom' imports the "useParams" hook to read from parametric URLs ("path='/route/:param'"), or any of the other routing hooks; using any one needed on any other components.

import queryString from 'query-string' imports the "queryString" object which allows to parse a query string.

<Switch>…</Switch> wraps the routes to be wired.

<Route>…</Route> defines the path to be evaluated and wraps the component to be rendered on that path. Has the following attributes:

  • "path": is set equal to the route to be associated with the component to be displayed ("path='/Page'"). A Route component without a "path" will render HTML code for a route not found.
  • "exact": empty attribute used to specify that the path must be matched exactly (no parametric URL).

<Redirect to='/Path' /> used inside a "<Route>" component to redirect to another path, where "to" can be an URL string or an object "{pathname: URL, search: ?queryString, state: { stateObject } }".

<Link to='/Path'>…</Link> creates a routable link (doesn't reload the whole application) to be used on rendered components.

useParams() returns the parameters from the URL ("let { param } = useParams()").

useLocation() returns an object that contains the location string, including the query string on its ".search" member.

useHistory() returns and object that allows to programmatically change routes from within our code. history.push(URLstring) method that allows to change route programmatically ("let history = useHistory()")

queryString.parse(location.search) parses the query string and returns an object made of the "key=value" pairs (the "location" object is returned by "useLocation()").
